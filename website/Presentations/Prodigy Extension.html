Anti-Cheat Shield

Browser-side exam proctoring. It assumes the client is hostile and the browser is not on your side. Nothing here is bulletproof — the goal is to raise the cost of cheating, log everything, and make tampering obvious.

03
// SECTION_01 · PREMISE

No proctor is bulletproof.

So this does not pretend to be. It is defense-in-depth running inside the student's browser: three layers that each fail differently, so a bypass of one does not mean a clean run.

01
// PREVENT

Friction

Fullscreen, tab-lock, DevTools, right-click and shortcut suppression. Not walls — speed bumps.

02
// DETECT

Signals

Every violation logs an integrity flag with timestamp + reason. Heuristics, not verdicts.

03
// ENFORCE

Consequence

Milestone lockouts, exit limits, penalty lockdowns — with a human appeal path.

READOUT

3/3

layers active. Remove one and the other two keep logging.

04
// SECTION_02 · TOPOLOGY

Six parts. Each distrusts the next.

content.js

Content Script

Injected into exam pages. Runs the monitors, lockout overlay, clock and appeal UI. The part users can inspect — so it verifies itself.

background.js

Service Worker

Owns exam mode, pulls remote config, brokers messages between tabs.

popup.html

Popup Monitor

Toolbar readout: mode, flag count, recent violations. Read-only.

server.js

Backend

Node server: config, sessions, lockouts, appeals, Hall of Fame. Source of truth.

admin.html

Admin Console

Live sessions, config, blocked users, appeal review — approve or deny.

config.json

Rule Set

One config, synced to every client. Milestones, exit caps, toggles.

// SECTION_03 · PARAMETERS

Tunable. Not magic.

Every threshold is a number an admin picks, then defends. Defaults are conservative on purpose — tighten them and you also raise false positives.

KEY DEFAULT EFFECT
milestoneInterval 5 flags per lockout
baseDurationSeconds 60 lockout length
maxExits 3 exits before penalty
requireFullscreen true force fullscreen
blockTabSwitch true flag focus loss
copyPasteAction toast toast / flag / block
CAVEAT

A stricter config is not a safer exam. It is a noisier one. Read the flags before you trust them.
