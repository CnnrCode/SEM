SLIDE 02 / OVERVIEW
§ 01 — Overview

What Is
SecureExam Browser?

SEB is not a conventional browser. It is a purpose-built kiosk application that locks the machine into a single-purpose exam session. The student cannot escape it through ordinary means.

Problem Statement

Online certification exams conducted via standard browsers are trivially cheatable. Tab-switching, screenshot tools, AI chat tabs, and clipboard abuse defeat honour-system policies within minutes. A policy is not an enforcement mechanism.

What SEB Does Instead
  • Forces kiosk-mode fullscreen — no chrome, no title bar
  • Intercepts and discards system-level hotkeys before the OS sees them
  • Blocks all known AI assistant domains at the network layer
  • Renders the window black in screenshots and screen recordings
  • Clears the system clipboard on session exit
  • Logs every suspicious event to a tamper-resistant audit file
Key Constraint

"No student input required to enforce security."

The system operates at the OS and network level — student cooperation is not assumed.

App Flow
  • First run → Admin setup
  • Configured → Kiosk exam
  • Ctrl+Shift+Alt+Q → Admin access
  • Exit → Password verify → cleanup

Process Isolation

Each module is a separate Node.js file loaded exclusively in the main process. The renderer communicates via Electron IPC — no raw Node access from the webpage context. contextIsolation: true is enforced.

Modules

SLIDE 04 / LOCKDOWN
§ 03 — Keyboard Lockdown

The Lockdown
System

lockdown.js registers ~40 system shortcuts through Electron's globalShortcut API — intercepting keystrokes before any other application or OS layer acts on them.

System Escape Vectors
  • Alt+Tab / Alt+Shift+Tab
  • Alt+F4
  • Ctrl+Alt+Delete
  • Ctrl+Shift+Escape
  • Win+D / Win+L / Win+R
  • Win+C (Copilot)
  • Win+G (Xbox Game Bar)
Screenshot / DevTools
  • PrintScreen
  • Alt+PrintScreen
  • Win+Shift+S (Snipping)
  • F12 (DevTools)
  • Ctrl+Shift+I / J / C
  • Ctrl+U (View Source)
  • Ctrl+F / F3 (Find)
Registry Enforcement (Windows)

SEB writes registry keys to HKCU\...\Policies\System to disable Task Manager at the OS level — beyond what globalShortcut can cover. Taskbar is hidden via StuckRects3. All restored cleanly on exit.

Two-Layer Defense

Layer 1 — Global Intercept

globalShortcut.register() claims the hotkey OS-wide. No other window receives it.

Layer 2 — Input Barrier

webContents.on('before-input-event') catches survivors and calls event.preventDefault().

SLIDE 05 / AI BLOCKING
§ 04 — Network Enforcement

AI Domain
Blocking

urlFilter.js enforces a deny-list at Electron's session layer. All URLs are allowed by default; only AI assistant domains are blocked — including all subdomains.

Subdomain Coverage

SLIDE 06 / SCREEN GUARD
§ 05 — Visual Security

Screen Guard &
Clipboard Control

screenGuard.js calls Electron's setContentProtection(true), which maps to SetWindowDisplayAffinity(WDA_EXCLUDEFROMCAPTURE) on Windows and CGWindowListSetWindowProperty on macOS. The exam window renders as solid black in any screenshot or screen recording.

Screen Capture Block
  • Win+PrintScreen → black
  • Snipping Tool → black
  • OBS / screen recorder → black
  • Shortcut also blocked at Layer 1
Clipboard Policy
  • Copy/paste disabled in-session
  • Clipboard wiped on exit
  • Right-click context menu blocked
  • Audit event logged on clear
Clipboard Poison (Advanced)

The clipboard is actively written with noise before wipe — preventing paste-before-clear attacks where a student copies exam content a fraction of a second before exit triggers.

Win32 API Layer

SetWindowDisplayAffinity

Called with WDA_EXCLUDEFROMCAPTURE. OS compositor excludes the window from any capture API — BitBlt, DWM, Game Bar.

Clipboard Wipe on Exit

clearClipboardOnExit triggers clipboard.clear() in the before-quit Electron lifecycle hook.

0 Pixels leaked to capture
SLIDE 07 / ADMIN
§ 06 — Administration

Admin Panel &
Configuration

A separate Electron window (admin.html) handles all runtime configuration. Access requires the admin password — salted SHA-256 stored in seb-config.json. On first run, setup is forced before the exam window opens.

Configurable Options
  • Block keyboard shortcuts
  • Block screen capture
  • Block copy/paste
  • Block right-click
  • Clear clipboard on exit
  • Block AI API backends
  • Block AI streaming
Access Control
  • Admin password (SHA-256 + salt)
  • Exit password (SHA-256 + salt)
  • Secret: Ctrl+Shift+Alt+Q
  • First-run wizard enforced
  • Config stored in userData
Password Storage
// config.js — password hashing function _hash(salt, password) { return crypto .createHash('sha256') .update(salt + password) .digest('hex'); } // Salt: crypto.randomBytes(16).toString('hex')
Security Notes

No plaintext passwords

Passwords are hashed before write. The JSON config never contains a recoverable secret.

No remote auth dependency

Auth is fully local. No network call is made — offline environments are supported.

Caution

If adminPasswordHash is empty, the system grants access. First-run enforcement is the guard against this.

SLIDE 08 / AUDIT TRAIL
§ 07 — Evidence Trail

Audit Log

Every security event is written to an append-only JSONL file in the Electron userData directory. The log survives the session and cannot be cleared through the normal exam flow.

Log Properties
Append-Only

Entries appended with fs.appendFileSync. No rotation. No truncation. History persists across sessions.

ISO 8601 Timestamps

UTC timestamps from new Date().toISOString() — directly correlatable with server-side exam timestamps.

JSONL Format

One JSON object per line. Parseable with jq, Python, or any log aggregator without special tooling.

SLIDE 09 / PLATFORMS
§ 08 — Platform Coverage

Multi-Platform
Deployment

Three distinct deployment targets, each matching a different institutional context. Feature parity is honest — not aspirational.

Feature Electron (Win) Android PWA
Kiosk fullscreen ✓ hard ✓ hard ~ soft
Hotkey block ✓ global partial none
AI domain block ✓ network WebView SW cache
Screen protection ✓ OS API FLAG_SECURE none
Audit log ✓ disk logcat none

PWA is the weakest link. It relies on browser-native fullscreen and Service Worker URL interception — both bypassable by opening another tab or using a different browser. Appropriate only for low-stakes scenarios or MDM-managed device pools.

Android Specifics
Kiosk Mode

FLAG_FULLSCREEN + SYSTEM_UI_FLAG_HIDE_NAVIGATION. Back button and recents button intercepted.

Screen Security

FLAG_SECURE on the window. Screenshots show black — equivalent to Electron's content protection.

PWA Anti-Cheat

Service Worker intercepts navigation. Visibility API triggers warnings on tab-switch. Policy-enforced, not technically enforced.

⚠ Known Gaps

  • External phone camera not blocked
  • PWA enforcement is weak
  • Admin bypass via registry edit
  • No biometric identity verification
  • No network traffic monitoring

SEB is a technical enforcement layer, not a complete proctoring solution. It raises the effort required to cheat by an order of magnitude — it does not eliminate the possibility. Pair with randomised question pools, time limits, and physical supervision for maximum effect.
